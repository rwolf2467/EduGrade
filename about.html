Security Features

EduGrade implements comprehensive security measures to protect against common web vulnerabilities, particularly Cross-Site Scripting (XSS) attacks. All user input is validated and sanitized before being displayed or stored.

XSS Protection (Cross-Site Scripting)

What is XSS?

XSS attacks occur when malicious code is injected into a website through user input. For example, if someone enters <script>alert('hack')</script> as a class name, without protection this code could execute in other users' browsers.

How EduGrade protects against XSS:

  • ✓ HTML Entity Encoding: Special characters like <, >, " are converted to harmless HTML entities (&lt;, &gt;, &quot;)
  • ✓ Pattern Detection: Input is scanned for dangerous patterns like <script>, javascript:, or event handlers like onclick=
  • ✓ Import Sanitization: All imported JSON data is thoroughly sanitized before use

Input Validation

All user inputs are validated before processing to ensure data integrity and prevent attacks:

Text Inputs

  • • Maximum length limits (50-200 chars)
  • • Empty input rejection
  • • Whitespace trimming
  • • Dangerous pattern detection

Numeric Inputs

  • • Grade values: 1-6 only
  • • Weights: 0.1-1.0 only
  • • NaN (Not a Number) rejection
  • • Type coercion handling

Security Headers (CSP)

Content Security Policy headers restrict what resources can be loaded, providing an additional layer of defense:

default-src 'self';

script-src 'self' cdn.jsdelivr.net;

style-src 'self' cdn.jsdelivr.net;

img-src 'self' data:;

This prevents loading scripts or styles from unauthorized sources, even if an attacker manages to inject HTML.

Data Encryption (AES-256-GCM)

Your data is encrypted with your password!

All user data (classes, students, grades) is encrypted using AES-256-GCM before being stored on the server. The encryption key is derived from your password - only you can decrypt your data.

How it works:

  • ✓ Key Derivation: Your password is used to derive a unique 256-bit encryption key using PBKDF2 with 100,000 iterations
  • ✓ AES-256-GCM: Military-grade authenticated encryption - data cannot be read OR tampered with
  • ✓ Unique Nonce: Each encryption uses a random 96-bit nonce for additional security
  • ✓ Server-side caching: Decrypted data is cached in memory during your session for performance, cleared when you close the page

Security Functions (security.js)

escapeHtml() Converts dangerous characters to HTML entities
validateStringInput() Validates text input for length and dangerous patterns
validateGradeValue() Ensures grades are valid (1-6 or +/-)
sanitizeImportData() Cleans all imported JSON data recursively
safeAttr() Creates safe values for HTML attributes

Latest Features

Student Access with PINs

Share grades securely with students using unique PINs. Each student gets their own 6-digit PIN to access their grades via a secure link. Configure what students can see and set expiration dates (1 hour to 30 days).

PDF Export & Print

Download and print detailed student views as PDF documents. Perfect for parent-teacher conferences or student reports. Includes all grades, charts, and statistics in a professional format.

CSV/JSON Student Import

Quickly import entire class rosters from CSV or JSON files. Save time when setting up new classes - no more manual student entry. Supports standard formats and validates data on import.

Subject-Based Organization

Organize classes into subjects (e.g., "Math", "English", "Science"). Each subject has its own categories and grades. A default subject is created automatically and can be renamed to fit your needs.

Detailed Student View with Charts

Click any student to see a comprehensive view with performance charts, grade history, category breakdowns, and statistical analysis. Visualize trends and identify areas for improvement at a glance.

Percentage-Based +/~/- System

The Plus/Minus/Neutral grading system now uses configurable percentage thresholds. Define exactly what percentage ranges correspond to +, ~, or - grades for fine-grained assessment.
